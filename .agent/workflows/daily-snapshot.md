## Daily Snapshot Workflow

1. Activate the virtual environment:
```
& c:/hrp-ops/.venv/Scripts/Activate.ps1
```

2. Run the full data rebuild (login, export, parse, fill tracker):
```
cd c:\hrp-ops; .\.venv\Scripts\python.exe scripts\02_export_stable.py
```

3. Parse and fill the tracker:
```
cd c:\hrp-ops; .\.venv\Scripts\python.exe scripts\04_parse_and_fill.py
```

4. Run the race results finish parser:
```
cd c:\hrp-ops; $env:PYTHONIOENCODING='utf-8'; .\.venv\Scripts\python.exe tmp_update_tracker.py
```

5. Run the tracker fix/enhancement script:
```
cd c:\hrp-ops; $env:PYTHONIOENCODING='utf-8'; .\.venv\Scripts\python.exe tmp_fix_tracker.py
```

6. Commit and push changes:
```
cd c:\hrp-ops; git add -A; git commit -m "Daily snapshot update"; git push
```

## Notes
- Step 2 requires a valid auth.json (run scripts\01_login_save_state.py first if expired)
- Step 2 takes ~45 minutes due to polite 12s delays between page fetches
- Reports (Dashboard, Weekly Plan, Decisions Log) are regenerated by the AI after the data rebuild
- SAFETY: No scripts submit forms that spend credits or change ownership
